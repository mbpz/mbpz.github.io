# 组件系统说明

## 概述

为了减少代码重复，项目使用了组件化设计。公共的HTML组件被提取到独立文件中，通过JavaScript动态加载。

## 组件结构

### 1. `components/sidebar.html`
包含侧边栏的完整HTML结构，包括：
- Logo（emo科技人卡通）
- 导航菜单

### 2. `js/components.js`
组件加载器，负责：
- 自动计算当前页面的相对路径
- 动态加载侧边栏组件
- 更新导航链接的href
- 设置当前页面的active状态

## 使用方法

在任何HTML页面中，只需：

1. 在 `<head>` 中引入CSS（根据页面层级调整路径）
2. 在 `<body>` 开始处添加注释：`<!-- 侧边栏将通过 components.js 动态加载 -->`
3. 在页面底部引入两个JavaScript文件（注意路径）：
   ```html
   <script src="[相对路径]js/components.js"></script>
   <script src="[相对路径]js/main.js"></script>
   ```

## 路径说明

组件系统会自动根据当前页面的路径深度计算正确的相对路径：
- 根目录：`./`
- 一级目录（如 `about/`）：`../`
- 二级目录（如 `2025/01/01/`）：`../../`
- 三级目录（如 `2025/12/12/`）：`../../../`

## 优势

1. **代码复用**：侧边栏代码只需维护一份
2. **易于更新**：修改侧边栏只需更新 `components/sidebar.html`
3. **自动路径处理**：无需手动计算每个页面的相对路径
4. **自动高亮**：当前页面会自动添加 `active` 类

## 注意事项

- 确保 `components.js` 在 `main.js` 之前加载
- 如果组件加载失败，会显示一个简单的备用侧边栏
- 所有导航链接使用 `data-page` 属性标识目标页面
